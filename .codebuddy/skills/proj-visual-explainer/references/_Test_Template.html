洞察卡片 (insight-item)

1 关注点分离

每个模块的提示词只教自己的事，上下游能力覆盖矩阵标注职责边界，避免能力重叠和提示词膨胀。

2 Pattern 驱动路由

同一意图可走不同版本。路由依据不是意图类别，而是提问句式是否命中一个有确定性处理方案的 Pattern。

3 渐进式披露

Base Prompt 只教 Agent 怎么想，EnhancedGrep 按需加载怎么写。上下文从 30K 降至 5K tokens。

SVG DIAGRAMS

嵌套 SVG 图表

流水线 SVG (pipeline-wrap)

使用 .pipeline-wrap 包裹 SVG。支持箭头 marker、圆角矩形、虚线路径、彩色文字、分支逻辑等。

架构 SVG (arch-diagram) — 组件关系图

使用 .arch-diagram 包裹。支持嵌套分组框、虚线背景区域、多色箭头、标注文字等复杂布局。

数据流 SVG — 交互链路图

支持多层分区、弯曲路径、注释标签、双向箭头等复杂交互。

CONTENT ELEMENTS

内容展示组件

提示框 (callout) — 7 种颜色

Accent：用于核心洞察、最重要的信息。边框颜色 var(--accent)。
Blue：用于技术说明、补充解释。边框颜色 var(--blue)。
Green：用于成功状态、最佳实践。边框颜色 var(--green)。
Amber：用于警告信息、注意事项。边框颜色 var(--amber)。
Red：用于错误提示、反模式。边框颜色 var(--red)。
Purple：用于战略方向、高层决策。边框颜色 var(--purple)。
Teal：用于基础设施、工具链信息。边框颜色 var(--teal)。
