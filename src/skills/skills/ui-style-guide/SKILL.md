## TASK
为 ChatCat 的 UI 改动提供统一视觉与交互标准，避免“每个面板一套风格”。

## STYLE TOKENS（全局基准）
- 字体：`'Segoe UI', 'Microsoft YaHei', sans-serif`
- 主容器：白底、深色描边、圆角、硬阴影
  - `background: #fff`
  - `border: 3px solid #222`
  - `border-radius: 22px`
  - `box-shadow: 3px 3px 0 #222`
- 主文本：`#222`；次要文本：`#888/#999`
- 主按钮：深色底（`#222`）+ 白字 + 轻微缩放 hover
- 危险态：`#f5576c`（关闭按钮 hover、错误提示等）

## PANEL 结构规范
任意弹层面板（聊天/工具/fun/快捷助手）默认结构：
1. Header（可拖拽，`-webkit-app-region: drag`）
2. Body（可滚动内容区）
3. Controls（`-webkit-app-region: no-drag`）
4. Bubble Tail（右下角尖角，指向猫咪）

要求：
- Header 内关闭/控制按钮必须是圆形描边按钮（24x24）
- Body 的输入框/结果框使用圆角 + 描边，避免玻璃拟态与主风格冲突
- 面板开关应支持互斥逻辑（同层窗口不叠开）
- 快捷助手（办公）底图必须与其他气泡面板一致，不能缺少右下角尖角

## 输入与结果区规范
- 输入框：
  - `background: #f8f8f8`
  - `border: 2px solid #222`
  - `border-radius: 12px~16px`
  - 聚焦色：`#0066cc`
- 结果区：
  - 浅灰背景（`#fafafa`）
  - 描边 `#e6e6e6`
  - 可滚动
  - 流式输出光标使用深色（与主风格一致）

## 按钮规范
- 主动作按钮（发送、保存、确认）：
  - 深色底 + 白字 + `font-weight: 700`
  - hover 时轻微放大（`transform: scale(1.03~1.05)`）
- 次动作按钮（历史、取消、反馈）：
  - 白/浅灰底 + 灰字
  - 不要用高饱和紫色等独立主题色，除非整站已切主题

## 交互一致性规范
- 工具栏入口按钮（聊天/生产力/快捷AI/fun）窗口需互斥：
  - 打开 A 时关闭 B/C/D
  - 再次点击当前入口关闭自身
- 快捷助手显示位置：
  - 每次打开都重新定位到猫猫上方
  - 超出屏幕时按工作区边界 clamp
- Tab 切换导致尺寸变化时，优先保持锚点稳定（避免压住猫）
- 拖动宠物时，快捷助手（办公）不应因主窗口重新聚焦而自动关闭
  - 仅在真正切出应用（焦点到外部程序）时才按自动隐藏规则关闭
  - 宠物拖动过程中的重定位应保持面板可见并持续跟随锚点

## 动画规范
- 面板出现：保留简短弹出动效（200~320ms）
- 尺寸变化：使用缓动（`cubic-bezier(0.22, 1, 0.36, 1)`）
- 仅在必要处添加动画，避免连续眩晕动效

## DO / DON'T
- DO：复用现有 `styles.css` 中的面板、按钮、输入框风格
- DO：新面板先对齐主面板，再做个性化微调
- DON'T：单独引入一套 glassmorphism 配色破坏统一性
- DON'T：让 Quick Panel 的按钮/标题/关闭控件脱离主规范
