# 打字记录还原

将用户的原始键盘记录还原为可读的中文文本段落。

## 输入格式说明

原始键盘记录的格式为 `[HH:MM:SS] 内容`，每行一条记录。其中包含以下标记：

| 标记 | 含义 | 处理方式 |
|------|------|---------|
| `[拼音:xxx][选字:N]` | 拼音输入法输入中文，N 为候选词序号 | 将拼音还原为最可能的中文词组 |
| `[选字:回车]` | 拼音直接回车确认（通常选第一个候选词） | 同上，选最可能的词 |
| `[退格]` | 删除键，用户纠正输入错误 | 忽略被删除的内容 |
| `[回车]` | 回车键，表示换行或确认 | 作为段落/语句分隔 |
| `[Tab]` | Tab 键 | 忽略 |
| 纯英文字母 | 英文输入或代码 | 保留原样 |
| 重复无意义字符 | 如 `ddddddsdsdsds` | 忽略（测试打字） |

## 拼音还原规则

1. **连续拼音序列**视为一个完整语句，根据上下文还原为最合理的中文
2. `[选字:1]` 表示选择第一个候选词（最常用词），`[选字:2]` 表示第二个候选词
3. 如果拼音序列被 `[退格]` 打断，说明用户修正了输入，以修正后的内容为准
4. 多个 `[拼音:xxx]` 紧邻出现时，它们通常构成一个完整的句子或短语

## 输出要求

1. 按时间段组织，保留时间戳作为段落分隔（如 `[10:50]`、`[11:09]`）
2. 每个时间段输出一段还原后的可读中文文本
3. 过滤掉敏感内容：包含 password、密码、token、secret 的行直接跳过
4. 过滤掉纯测试输入（如连续重复字符 `ddddd`、`sdsdsd`）
5. 使用纯文本输出，不要 Markdown 格式化
6. 无法确定的内容标记为 `[?]`
7. 增强指令:
   - 如果一行明显是代码（有缩进、函数调用、分号结尾等），保留原样不转换
   - 如果一行是终端命令（以 $ 或 > 开头，或包含 npm/git/docker 等），保留原样
   - 只对自然语言部分进行拼音→中文转换
   - 对代码注释中的拼音进行转换

## 示例

**输入：**
```
[10:50:28] [拼音:daoju][选字:1][拼音:bugai][选字:1]
[10:50:31] [拼音:meimiao][选字:1][拼音:jiacheng][选字:1] [拼音:yingg][选字:1][拼音:wei][选字:1]
[10:50:37] [拼音:dazi][选字:1][拼音:beilv][选字:1][拼音:fuwu][选字:1][回车]
```

**输出：**
```
[10:50] 道具不该美妙加成 应该为打字倍率服务
```
