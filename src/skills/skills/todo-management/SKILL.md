# 待办提取

从用户今日打字记录中识别待办事项，与已有待办对比后输出**仅新增的**待办。

## 数据来源

- **convertedText**：已转换的可读中文文本（优先使用）
- **rawTyping**：原始键盘记录（兜底，需将拼音还原为中文）
- **todos**：用户已有的待办列表，**必须对比去重**
- **userMessage**：用户可能直接输入了待办内容（如 `/todo 写周报`）

## 提取规则

只提取高置信度的待办线索：

| 模式 | 示例 |
|------|------|
| 明确任务 | "需要完成xxx"、"要做xxx"、"改一下xxx" |
| 有时间约束 | "明天要xxx"、"周五前完成"、"下周需要" |
| 承诺/约定 | "答应了xxx"、"和xxx约好了" |
| 自我提醒 | "记得xxx"、"别忘了xxx" |
| 问题待解决 | "这个bug需要修"、"还没搞定" |

**不要提取的内容：**
- 普通陈述句（"今天天气不错"）
- 已完成的事情（"刚才已经修好了"）
- 与已有待办重复或相似的内容

## 去重规则

对比已有 todos 列表：
- 语义相同的不重复添加（如已有"写周报"，不再添加"完成周报"）
- 只输出**真正新增的**待办

## 输出格式

每个待办项 MUST 使用这个格式（系统会自动解析）：

```
- [ ] 待办内容
```

示例：

```
- [ ] 修复登录页面报错
- [ ] 完成本周周报
- [ ] 和产品讨论新功能方案
```

如果没有新的待办，输出：

```
没有发现新的待办事项。
```
