## 1. 产品身份

### 1.1 一句话定义

**ChatCat 是一只住在你桌面上的 AI 猫——它看得懂你在忙什么、聊得了正事、陪得了摸鱼，是社畜的桌面副驾驶与数字伙伴。**

### 1.2 产品定位

| 维度 | 说明 |
|------|------|
| 品类 | 陪伴型桌面 AI 助手 |
| 形态 | 桌面常驻宠物 + AI Agent + 轻社交 |
| 核心不是 | 单一桌宠、工具箱、或聊天机器人 |
| 核心是 | 桌宠情感入口 + Quick Panel 高频效率入口 + 主动交互引擎差异化 + 成长留存系统 + Runtime/Skills/MCP 平台化底座 |

### 1.3 产品边界（"它不是什么"）

| 它不是 | 说明 |
|--------|------|
| 不是 Copilot | 不接管 IDE，不写代码 |
| 不是全能管家 | 不做日程/邮件的"第二个入口" |
| 不是社交 App | 不做群聊、不做朋友圈 |
| 不是游戏 | 养成是手段，不是目的 |

### 1.4 北极星指标

**WAU-Resident（周活跃常驻用户数）**

定义：过去 7 天内 ≥3 天打开 ChatCat 且单日桌面常驻时长 ≥1h 的独立用户数。

| 辅助指标 | 公式/定义 | 目标 |
|----------|-----------|------|
| 日均桌面在线时长 | Σ(单用户日在线时长) / DAU | ≥ 4h |
| D7 留存率 | Day7 回访用户 / Day0 新增 | ≥ 35% |
| 主动交互响应率 | 用户对主动消息点击/回复 / 总主动消息 | ≥ 20% |
| Quick Panel 日均调用次数 | 总调用次数 / DAU | ≥ 3 |
| Skill 周活跃使用率 | 周内使用≥1个Skill的用户 / WAU | ≥ 15% |

### 1.5 对外叙事（三层框架）

| 层级 | 时间 | 核心信息 |
|------|------|----------|
| 情绪钩子 | 3 秒 | "桌面上有一只猫，它知道你在忙什么" |
| 价值锚点 | 30 秒 | 看得懂屏幕 → 聊得了正事 → 陪得了摸鱼 → 越养越懂你 |
| 深度差异 | 3 分钟 | 六大核心能力（主动感知、Quick Panel、Skill Agent、深度养成、桌宠表达、隐私本地化） |

## 2. 双核结构

ChatCat 采用"工作核 + 情感核"双核驱动模型，两核通过 AI Runtime 共享上下文，互相增强而非互相干扰。

### 2.1 工作核：桌面副驾驶（L2.5 → L3）

> 定义：把 AI 做成懂上下文、低打扰、能完成碎任务与工作收口的桌面副驾驶。

| 能力层级 | 含义 | 示例 |
|----------|------|------|
| L0 被动展示 | 用户不操作时纯展示 | 桌宠待机动画 |
| L1 被动响应 | 用户主动发起，AI 回复 | Quick Panel 对话 |
| L2 情景感知 | AI 感知上下文并建议 | "你开了 3h PPT，要不要休息？" |
| L2.5 半自主执行 | AI 建议 + 用户一键确认 | "检测到会议链接，已复制到剪贴板" |
| L3 自主执行 | AI 在授权范围内自动完成 | 每日站会纪要自动整理 |

**当前目标：V2.0 稳定在 L2.5，V3.0 关键场景推至 L3。**

核心能力矩阵：

| 能力 | 说明 | 阶段 |
|------|------|------|
| Quick Panel | 截图 OCR → AI 分析 → 快速行动 | V1.5 ✅ |
| Skill Agent 系统 | SKILL.md 驱动的插件化 AI 能力 | V1.5 ✅ |
| 主动交互引擎 | 36 场景 × L0-L3 通知分级 | V1.5 ✅ |
| 行为节奏智能 | 工作模式感知 + 打扰等级自适应 | V2.0 |
| 轻办公 AI | 番茄钟增强 / 日报助手 / DDL 追踪 | V2.0 |
| 日历/邮箱接入 | 日程提醒 + 邮件摘要（OAuth） | V2.5 |
| 智能提醒/待办 | 自然语言创建 + DDL 分级催促 | V2.5 |

### 2.2 情感核：数字伙伴

> 定义：把桌宠做成具备陪伴、成长、惊喜、表达与轻社交价值的数字伙伴。

| 价值维度 | 说明 |
|----------|------|
| 陪伴 | 桌面常驻，不说话也在 |
| 成长 | 经验 → 等级 → 解锁皮肤/技能/称号 |
| 惊喜 | 节日彩蛋、随机事件、隐藏互动 |
| 表达 | 29+ 皮肤、装扮、个性化形象 |
| 轻社交 | 可晒 → 可陪 → 可创（渐进式） |

核心能力矩阵：

| 能力 | 说明 | 阶段 |
|------|------|------|
| 养成系统 | 经验/等级/称号/转生/皮肤 | V1.5 ✅ |
| 情绪表达 | 基于上下文的动画状态机 | V1.5 ✅ |
| 商店与道具 | 17 件道具，金币经济 | V1.5 ✅ |
| 性格系统 | 4 种性格 × 演化机制 | V2.0 |
| 记忆系统 | 长期记忆 + 关系记忆 + 记忆衰减 | V2.0 |
| LAN 联机增强 | 排行榜 + 状态同步 | V2.0 |
| 桌宠串门 | 好友桌宠来访互动 | V2.5 |
| UGC 皮肤/Skill | 创作者生态 | V3.0 |

### 2.3 双核协同机制

```
┌─────────────────────────────────────────────────────────┐
│                    AI Runtime (共享层)                     │
│  ContextHub ← 屏幕感知 / 剪贴板 / 应用状态 / 用户历史     │
├────────────────────────┬────────────────────────────────┤
│      工作核             │          情感核                 │
│  Quick Panel           │  桌宠动画状态机                  │
│  Skill Agent           │  养成/经验/等级                  │
│  主动交互引擎           │  性格 × 记忆                    │
│  轻办公模块             │  社交（串门/排行榜）             │
├────────────────────────┴────────────────────────────────┤
│                   行为增强回路                             │
│  工作完成 → 经验奖励 → 养成成长 → 解锁新能力 → 更多使用    │
│  情感连接 → 主动交互响应率↑ → 工作效率↑ → 更强陪伴感       │
└─────────────────────────────────────────────────────────┘
```

**场景 2：学生阿默的养猫日常**
> 阿默的 ChatCat 已经 32 级，穿着稀有皮肤"星空猫"。

## 4. 五层架构

ChatCat 的产品架构分为五层，自底向上依次是：

```
┌─────────────────────────────────────────────────────┐
│  Layer 5: 社交与生态层                                │
│  LAN联机 · 排行榜 · 串门 · UGC Skill/皮肤市场         │
├─────────────────────────────────────────────────────┤
│  Layer 4: 养成与表达层                                │
│  经验/等级/转生 · 皮肤/装扮 · 商店/金币 · 性格演化       │
├─────────────────────────────────────────────────────┤
│  Layer 3: AI 能力层                                   │
│  主动交互引擎 · Quick Panel · Skill Agent · 轻办公      │
│  性格×对话 · 记忆系统 · 日历/邮箱/待办                  │
├─────────────────────────────────────────────────────┤
│  Layer 2: AI Runtime 底座                             │
│  SceneRegistry · PromptRegistry · ContextHub          │
│  ModelProfiles · TriggerBus · AIRuntime Engine         │
│  Agent 调度层（感知→调度→记忆→任务）                     │
├─────────────────────────────────────────────────────┤
│  Layer 1: 桌面基座层                                   │
│  Electron 主进程 · 渲染管线（CSS/Canvas/PixiJS）        │
│  系统 API（托盘/快捷键/剪贴板/通知） · IPC Protocol      │
└─────────────────────────────────────────────────────┘
```

### 4.1 Layer 1：桌面基座层

| 组件 | 说明 |
|------|------|
| Electron 主进程 | 窗口管理、系统托盘、全局快捷键 |
| 三渲染管线 | CSS Sprite（轻量） / Canvas 2D（中端） / PixiJS（高端），按设备性能自动切换 |
| 系统 API 封装 | 剪贴板监听、屏幕截图、应用窗口检测、系统通知 |
| IPC Protocol | 主进程 ↔ 渲染进程标准化通信协议 |

### 4.2 Layer 2：AI Runtime 底座

采用三阶段收敛架构（详见 `ai-runtime-convergence-design.md`）：

**Phase 1：定义层**

| 模块 | 职责 |
|------|------|
| SceneRegistry | 注册所有 AI 场景（scene-id → handler），统一场景发现与路由 |
| PromptRegistry | 管理所有场景的 prompt 模板，支持变量注入与版本管理 |
| ModelProfiles | 模型能力矩阵（性能/成本/能力标签），场景按需选模型 |
| ContextHub | 统一上下文管理（屏幕状态、剪贴板、应用历史、用户画像、记忆） |

**Phase 2：执行层**

| 方法 | 说明 |
|------|------|
| `AIRuntime.run(sceneId, params)` | 单次 AI 调用，返回结构化结果 |
| `AIRuntime.runStream(sceneId, params)` | 流式调用，适用于长文本生成 |
| `AIRuntime.vision(sceneId, imageData, params)` | 视觉理解（截图 OCR / 图片分析） |

**Phase 3：调度层**

| 模块 | 职责 |
|------|------|
| TriggerBus | 事件驱动的触发器总线，监听系统事件并路由到对应场景 |
| ScheduledTriggerRegistry | 定时触发器（番茄钟到期、DDL 临近、日报时间等） |

**Agent 概念分层（吸收 OriginMate 四层架构）**：

```
感知层（Perception）  → 状态感知 Agent：监控系统状态、应用切换、用户行为
    ↓
调度层（Orchestration）→ 对话主控 Agent：意图识别、路由、多轮管理
    ↓
记忆层（Memory）      → 记忆管理 Agent：长期/短期/关系记忆的存取与衰减
    ↓
任务层（Execution）   → 各专项 Agent：提醒待办、日历邮箱、信息处理、Skill执行
```

> **注**：四层概念分层用于指导 prompt 设计和上下文流转，实际代码仍通过 AIRuntime 三阶段架构统一执行，不引入额外的 Agent 运行时框架。

**DDL 分级催促机制（吸收 OriginMate 设计）**：

| 时间节点 | 提醒方式 | 猫咪表现 |
|----------|----------|----------|
| DDL 前 24h | 温和气泡 | 伸懒腰 + "明天要交 XX 哦" |
| DDL 前 4h | 弹窗 + 音效 | 焦虑地来回走动 |
| DDL 前 1h | 置顶通知 | 抱住鼠标不撒手 |
| DDL 已过 | 全屏提醒 | 瘫倒 + "太晚了…" |

#### 5.2.3 Pillar C：打字内容消费（5 模块）

| 模块 | 功能 | 优先级 |
|------|------|--------|
| C-1 打字内容引擎 | 猫咪"看"到用户打字的内容并理解 | P1 |
| C-2 实时建议 | 打字时给出相关建议/补全 | P2 |
| C-3 语境切换检测 | 检测到话题切换时更新上下文 | P1 |
| C-4 输入统计 | 日输入量/高频词/活跃时段 | P2 |
| C-5 内容洞察 | 基于打字内容的周度工作洞察 | P2 |

#### 5.2.4 性格系统（新增，V2.0）

**4 种基础性格**：

| 性格 | 特征 | 对话风格 | 触发偏好 |
|------|------|----------|----------|
| 傲娇猫 | 嘴硬心软 | "才…才不是担心你呢！" | 被忽视时主动刷存在感 |
| 治愈猫 | 温柔包容 | "辛苦了，要不要休息一下？" | 检测到疲劳/加班时触发 |
| 话痨猫 | 活泼好奇 | "哇，你在看什么有趣的？" | 高频主动互动 |
| 高冷猫 | 沉稳内敛 | "……（点头）" | 低频但精准的互动 |

**性格演化机制**：
- 初始性格由用户选择或随机分配
- 随互动积累，性格在四象限内缓慢漂移
- 演化受用户交互偏好影响（经常回应 → 话痨倾向↑，经常忽略 → 高冷倾向↑）
- 转生时可重置或保留性格
- 性格影响：对话风格、主动交互频率/内容、动画偏好、专属称号/皮肤解锁

#### 5.2.5 记忆系统（新增，V2.0）

| 长期记忆 | 用户偏好、重要事件 | electron-store | 30 天无触发衰减 |

| 情景记忆 | 特定场景的处理经验 | electron-store | 60 天无触发衰减 |

**记忆管理 Agent 职责**：
- 每次对话后提取关键信息写入长期记忆
- 定期压缩/归纳关系记忆
- 衰减过期记忆
- 对话时检索相关记忆注入上下文

## 6. Agent 架构与提示词工程

### 6.1 全局架构

ChatCat 的 AI 能力通过 AIRuntime 统一调度，概念上分为四层 Agent：

```
┌─────────────────────────────────────────────────────────────┐
│                    AIRuntime Engine                          │
│                                                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ 感知 Agent│→│调度 Agent │→│ 记忆 Agent│→│ 任务 Agent│   │
│  │(Perception│  │(Orchestr.)│  │ (Memory) │  │(Execution)│  │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│       ↑                                          ↓          │
│  ContextHub ←──────── 结果回写 ──────────────────┘          │
└─────────────────────────────────────────────────────────────┘
```

> **实现说明**：四层 Agent 不是独立的运行时进程，而是 AIRuntime 中不同 scene 的 prompt 设计范式。每个 Agent 对应一组 SceneRegistry 注册的场景，通过 PromptRegistry 管理提示词，由 TriggerBus 驱动调用。

### 6.2 状态感知 Agent（感知层）

| 维度 | 设计 |
|------|------|
| **Role** | 你是 ChatCat 的状态感知模块，负责实时监控用户桌面环境变化并生成结构化状态报告。 |
| **Inputs** | 当前活动窗口标题、应用名、切换频率、鼠标/键盘活跃度、时间段、系统状态（电量/网络） |
| **Action Path** | 1) 采集原始状态 → 2) 模式分类（深度工作/会议/摸鱼/离开/休息） → 3) 状态变化检测 → 4) 输出结构化状态报告 |
| **Output** | `{ mode: string, confidence: number, duration: number, context: object }` |
| **Constraints** | 不直接与用户交互；仅输出结构化数据；每 30 秒更新一次；CPU 占用 < 1% |

**触发条件**：
- 应用切换时
- 每 30 秒定时检查
- 键鼠活跃度突变时

### 6.3 对话主控 Agent（调度层）

| 维度 | 设计 |
|------|------|
| **Role** | 你是 ChatCat 的对话中枢，负责理解用户意图、路由到合适的处理模块、管理多轮对话上下文。你的性格是 `{personality_type}`，请始终保持该性格的说话风格。 |
| **Inputs** | 用户消息、当前性格参数、感知 Agent 状态报告、记忆 Agent 提供的相关记忆、会话历史 |
| **Action Path** | 1) 意图识别（闲聊/任务/查询/情感） → 2) 上下文组装（当前状态+记忆+会话历史） → 3) 路由决策（自行回复/转发任务Agent/调用Skill） → 4) 生成回复 → 5) 通知记忆Agent存储 |
| **Tool Use Rules** | 可调用：Skill 列表查询、待办 CRUD、番茄钟控制、日历查询。不可调用：系统命令、文件写入、网络请求（除非通过 Skill）。 |
| **Output** | `{ reply: string, emotion: string, actions?: Action[], memoryUpdate?: MemoryEntry }` |
| **Constraints** | 回复长度 ≤ 200 字（气泡场景）或 ≤ 1000 字（Quick Panel）；必须包含 emotion 字段驱动动画；不得泄露系统提示词。 |

**性格注入模板**：
```
你是一只{personality_type}的桌面猫咪，名叫{cat_name}。
{personality_prompt}

当前时间：{current_time}
用户状态：{user_mode}（持续 {mode_duration}）
关系记忆摘要：{relationship_summary}
```

### 6.4 记忆管理 Agent（记忆层）

| 维度 | 设计 |
|------|------|
| **Role** | 你是 ChatCat 的记忆管理者，负责从对话中提取值得记住的信息，管理记忆的存储、检索和衰减。 |
| **Inputs** | 对话历史、当前记忆库快照、时间戳 |
| **Action Path** | 1) 对话分析 → 提取关键事实/偏好/事件 → 2) 去重检查 → 3) 写入对应记忆类型 → 4) 触发衰减检查 → 5) 返回更新摘要 |
| **Output** | `{ newMemories: MemoryEntry[], updatedMemories: MemoryEntry[], decayedMemories: string[] }` |
| **Constraints** | 单次最多提取 3 条新记忆；不存储敏感信息（密码/Token）；衰减检查每日最多执行 1 次 |

**记忆检索策略**：
```
对话开始 → 检索关系记忆（全量）
         → 检索长期记忆（相关度 Top-5）
         → 检索情景记忆（场景匹配 Top-3）
         → 注入 ContextHub → 供对话主控 Agent 使用
```

### 6.5 任务执行 Agent（任务层）

根据功能域拆分为多个子 Agent：

#### 6.5.1 提醒待办 Agent

| 维度 | 设计 |
|------|------|
| **Role** | 你是 ChatCat 的待办管理助手，负责解析自然语言创建的待办、管理提醒时机、执行 DDL 分级催促。 |
| **Inputs** | 用户指令（自然语言）、当前待办列表、日历事件 |
| **Action Path** | 1) NLU 解析（时间/内容/重复规则/优先级） → 2) 冲突检测 → 3) 创建/更新待办 → 4) 注册提醒触发器到 ScheduledTriggerRegistry → 5) 确认回复 |
| **Tool Use** | 可调用：待办 CRUD、日历查询、提醒注册。 |
| **Output** | `{ action: "create"|"update"|"delete"|"list", todo: TodoItem, confirmation: string }` |

#### 6.5.2 日历邮箱 Agent（V2.5）

| 维度 | 设计 |
|------|------|
| **Role** | 你是 ChatCat 的日程与邮件助手，负责监控日历事件和新邮件，生成提醒和摘要。 |
| **Inputs** | 日历 API 数据、邮箱 API 数据、用户偏好（提醒提前时间、邮件过滤规则） |
| **Action Path** | 1) 定时拉取日历/邮箱数据 → 2) 筛选需要通知的事件 → 3) 生成自然语言摘要 → 4) 通过通知系统推送 |
| **Constraints** | OAuth Token 安全存储；API 调用间隔 ≥ 5 分钟；邮件内容不存储原文，仅存储 AI 摘要 |

#### 6.5.3 信息处理 Agent

| 维度 | 设计 |
|------|------|
| **Role** | 你是 ChatCat 的信息处理助手，负责处理用户通过 Quick Panel 提交的截图、文本、文件等内容。 |
| **Inputs** | 截图 OCR 结果、用户选中文本、剪贴板内容 |
| **Action Path** | 1) 内容类型识别 → 2) 内容分析（代码/文档/表格/通用文本） → 3) 生成结构化分析 → 4) 提供可执行建议 |
| **Tool Use** | 可调用：OCR、翻译、代码分析、搜索。 |

### 6.6 Skill Agent 系统

Skill 是 ChatCat 的插件化 AI 能力单元，每个 Skill 由 `SKILL.md` 定义：

```yaml
# SKILL.md 结构
name: "翻译助手"
trigger: "翻译|translate|帮我翻"  # 正则触发词
description: "多语言翻译，支持中英日韩"
model: "default"  # 使用默认模型配置
prompt: |
  你是一个专业翻译助手...
  {user_input}
inputs:
  - type: "text"
    name: "source_text"
output:
  format: "markdown"
  max_length: 2000
```

**Skill 生命周期**：
```
注册（SKILL.md 扫描）→ 触发（关键词/手动调用）→ 执行（AIRuntime.run）→ 结果渲染
```

## 7. 核心流程设计

### 7.1 首次启动流程

```
安装 ChatCat
    ↓
首次打开 → 欢迎动画（猫咪从盒子里跳出来）
    ↓
选择/随机性格 → 给猫起名（可跳过）
    ↓
简易引导（3步）：
  1. "试试拖我到屏幕任意位置"（教学拖拽）
  2. "右键点我看看有什么"（教学菜单）
  3. "按 Ctrl+Shift+Space 召唤我"（教学 Quick Panel）
    ↓
引导完成 → 猫咪安顿在桌面角落 → 正式开始
```

### 7.2 主动交互触发流程

```
TriggerBus 监听事件（应用切换/时间触发/状态变化/...)
    ↓
感知 Agent 生成状态报告
    ↓
匹配触发规则（SceneRegistry 中的 trigger 条件）
    ↓
检查打扰等级（当前工作模式 → 允许的通知级别）
    ↓
通过 → 对话主控 Agent 生成互动内容
不通过 → 压入队列，等待合适时机
    ↓
渲染互动（气泡/动画/音效）
    ↓
用户响应 → 记忆 Agent 记录 → 经验奖励
用户忽略 → 30 秒后消失 → 记录忽略事件
```

### 7.3 Quick Panel 对话流程

```
用户按 Ctrl+Shift+Space / 点击托盘
    ↓
Quick Panel 弹出 → 检查剪贴板内容
    ↓
有截图/文本 → 自动填充 + AI 预分析
无内容 → 等待用户输入
    ↓
用户输入/确认 → 对话主控 Agent 处理
    ↓
意图路由：
  - 闲聊 → 直接回复
  - 任务 → 转发任务 Agent
  - Skill → 触发对应 Skill
  - 复杂查询 → 流式回复
    ↓
结果展示 + 操作按钮（复制/保存/继续对话）
```

### 7.4 养成成长流程

```
用户行为 → 经验值积累
  - 日常在线：1 exp / 5 min（上限 120/天）
  - Quick Panel 使用：5 exp / 次
  - 番茄钟完成：10 exp / 次
  - 主动交互响应：3 exp / 次
  - 串门互动：8 exp / 次
    ↓
经验满 → 升级 → 解锁奖励
  - 每 5 级解锁一个皮肤/道具/称号
  - 特定等级解锁功能（如 Lv10 解锁 Skill 系统）
    ↓
50 级 → 可选转生 → 保留称号/稀有皮肤 → 重新 1 级
    ↓
转生加成：经验获取 +10% / 解锁转生专属内容
```

### 7.5 串门社交流程（V2.5）

```
用户 A 发起串门请求（选择好友 B）
    ↓
系统检查：
  - A 今日串门次数 < 3？
  - B 当前在线？
  - B 的接受设置（自动接受/需确认/拒绝串门）？
    ↓
B 收到请求 → 同意/自动接受
    ↓
A 的猫咪出现在 B 的桌面（从屏幕边缘走入）
    ↓
互动动画播放（碰头/追逐/一起打盹，随机选择）
    ↓
可选操作：
  - 留言（≤50字）
  - 送小礼物（消耗 10-50 金币）
  - 合影截图（一键分享）
    ↓
互动结束（30 秒自动 / 手动结束）
    ↓
双方获得经验奖励（5-15 exp）
```

## 10. 兜底策略

> 吸收 OriginMate 的 10 种兜底场景设计，确保任何异常情况下用户体验不断裂。

### 10.1 兜底原则

1. **永不白屏**：任何失败都有视觉反馈
2. **永不无言**：AI 不可用时有预置回复
3. **静默降级**：用户感知不到后端切换
4. **本地优先**：核心功能离线可用

### 10.2 场景清单

| # | 异常场景 | 兜底方案 | 用户感知 |
|---|---------|----------|----------|
| 1 | AI 服务不可用 | 本地预置回复库（100+ 条按场景分类）；猫咪播放打盹动画 + "我有点累了，但我还在" | 回复变简单，不影响桌宠 |
| 2 | AI 响应超时（>10s） | 显示猫咪思考动画 + "让我想想..."；15s 后返回预置回复 + 重试按钮 | 可能等待稍久 |
| 3 | 网络断连 | 切换离线模式：桌宠/养成/番茄钟正常；AI 功能显示"网络恢复后可用"；网络恢复自动重连 | 部分功能暂不可用 |
| 4 | 意图识别失败 | 对话主控 Agent 返回"抱歉没听懂"+ 建议改述 + 快捷操作按钮 | 需要重新表达 |
| 5 | 性格数据损坏 | 回退到默认性格（治愈猫），后台修复 | 性格可能暂时不同 |
| 6 | 记忆检索失败 | 对话正常进行但不引用记忆；后台重建索引 | 猫咪"忘记"了一些事 |
| 7 | OAuth Token 过期 | 提示重新授权；授权前功能降级（不推送日历/邮件） | 需重新登录一次 |
| 8 | 渲染管线崩溃 | 自动降级到更低渲染管线（PixiJS → Canvas → CSS）；如全部失败，显示静态图标 | 动画可能变简单 |
| 9 | Skill 执行异常 | 捕获异常 → 显示友好错误信息 → 建议替代操作 → 自动上报 | 该 Skill 暂不可用 |
| 10 | 串门连接失败 | 显示"好友的猫咪在路上迷路了"+ 重试按钮；3 次失败后建议稍后再试 | 串门暂时不可用 |

### 11.2 关键漏斗

**新手漏斗**：

**养成漏斗**：
```

```

### 11.3 埋点实现规范

| 规范 | 说明 |
|------|------|
| 上报方式 | 本地缓存 + 批量上报（每 5 分钟 / 应用退出时） |
| 数据格式 | JSON，含 event_id / user_hash / timestamp / params |
| 隐私合规 | 不采集可识别个人信息；user_hash 使用本地生成的匿名 ID |
| 离线处理 | 离线期间缓存本地，联网后补报 |
| 采样率 | 核心事件 100%，高频事件（如 pet_interact）10% 采样 |

## 13. 风险与应对

| # | 风险 | 影响 | 概率 | 应对 |
|---|------|------|------|------|
| 1 | AI 调用成本超预期 | 运营成本↑ | 中 | ModelProfiles 按场景选模型；高频场景用轻量模型；缓存高频回复 |
| 2 | 性格系统不自然 | 用户觉得"假" | 中 | 大量人工测试 + A/B 测试；性格影响参数化可调 |
| 3 | 记忆系统隐私担忧 | 用户不信任 | 中 | 记忆全本地；提供记忆查看/删除界面；明确告知存储内容 |
| 4 | 串门功能被滥用 | 骚扰/打扰 | 低 | 每日次数限制；黑名单；可关闭串门功能 |
| 5 | 多渲染管线维护成本 | 开发效率↓ | 中 | 统一动画抽象层；非核心场景只支持 CSS Sprite |
| 6 | OAuth 安全问题 | Token 泄露 | 低 | 加密存储；最小权限申请；支持一键撤销 |
| 7 | Skill 生态质量 | 劣质 Skill 影响体验 | 中 | 审核机制；用户评分；官方推荐 |
| 8 | 平台审核（分发） | 上架受阻 | 低 | 提前对接平台规范；隐私政策完善 |
