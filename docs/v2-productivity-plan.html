设计原则：零内容采集

ChatCat 灵感来自 BongoCat，原项目只采样键盘和鼠标的操作频率。
V2 严格遵循这一原则：不读键盘内容、不读剪贴板文本、不读窗口标题。
所有 Pillar A 的智能能力，完全基于以下无内容数据：

数据源采集内容是否涉及内容
键盘 keycode击键频率、速度(CPM)、节奏、退格率❌ 纯频率
鼠标 click点击频率、按钮类型❌ 纯事件
鼠标 mousemove移动距离、速度❌ 纯坐标
时间小时、星期、工作时长❌ 纯时钟

用户确认的需求

Pillar A: 行为节奏可视化 个性化猫咪反应 鼠标信号化 AI节奏日报/周报
Pillar B: 润色/改写 总结/提取 解释/教学 截图OCR 快捷问答

忠于BongoCat · 只用键鼠频率 + 时间 · 零内容采集 · 零隐私风险

🔒 核心原则：只读频率，不读内容

不读键盘内容（无 RecentTypingBuffer）· 不读剪贴板文本 · 不读窗口标题

Pillar C: 打字内容消费 (5个模块) 用户授权可选

基于打字数据的下游智能消费 · 默认关闭 · 安全协议授权后开启 · 数据清洗→消费

🔐 核心原则：用户知情授权 + 数据先清洗再消费

Pillar A/B 完全不依赖打字内容，Pillar C 是独立的可选分支。
用户未授权时，Pillar C 所有模块不加载、不采集、不消耗任何资源。
用户授权后，打字内容先经过「数据清洗管道」过滤敏感信息，再进入下游消费。
数据流: 原始按键 → 敏感过滤 → 拼音转换 → 内容分段 → 日报/待办/回顾

C1
敏感信息过滤器 Sensitive Filter
src/cleaner/sensitive-filter.js 主进程 新建

数据清洗管道的第一步。在原始按键记录写入文件之前，先过滤掉敏感行。

过滤规则匹配模式处理方式
密码行包含 password / 密码 / pwd整行替换为 [FILTERED]
令牌行包含 token / secret / api_key / apikey整行替换为 [FILTERED]
纯密文行全是 • / * / ● 的行整行丢弃
长无意义串连续40+字符无空格（可能是hash/密钥）截断并标记 [HASH]
// 过滤前 "export const API_KEY = 'sk-abc123def456...'" // 过滤后 "[FILTERED: 包含敏感关键词 api_key]"

原则: 宁可多过滤，不可少过滤。误伤率可接受，泄露不可接受。

内容分段器 Content Segmenter
src/cleaner/content-segmenter.js 主进程 新建

清洗管道的最后一步：将已转换的干净文本按时间和类型分段，为下游消费提供结构化数据。

分段维度判断依据输出标签
时间段打字暂停 >5min 为分段点segment.startTime / endTime
内容类型代码特征(缩进/符号) vs 自然语言type: code | text | chat
语言中英文比例lang: zh | en | mixed
密度字符数 / 时间density: high | medium | low
// 分段器输出 [{ startTime: '10:05', endTime: '10:42', type: 'code', lang: 'en', density: 'high', charCount: 2840, summary: '连续编码37分钟' }, { startTime: '11:00', endTime: '11:15', type: 'text', lang: 'zh', density: 'medium', charCount: 580, summary: '写了一段中文文档' }]
