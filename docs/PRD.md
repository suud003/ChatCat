# ChatCat PRD - AI 主动引导型桌面宠物 + 智能办公助手

## 1. 产品概述

### 1.1 产品定位
ChatCat 是一款 **AI 主动引导型** 桌面宠物+智能办公助手。猫咪会 **主动观察你的工作状态、主动整理信息、主动在合适时机提供帮助**。用户只需在桌面养一只猫——它会自己学习你的习惯，在你需要时出现，在你专注时安静陪伴。

不同于传统的"工具箱"式办公软件，ChatCat 以 **猫咪为主体、AI 为大脑**，将信息整理、情绪关怀、效率引导融入一只有性格、有记忆、会成长的虚拟猫咪中。用户无需主动打开任何功能面板——猫咪会在合适的时机，用合适的方式，主动把帮助送到你面前。

### 1.2 目标用户
- **核心用户**: 每天使用电脑 6h+ 的知识工作者、程序员、学生
- **扩展用户**: 喜欢桌面美化/虚拟宠物的休闲用户
- **画像**: 注重效率但希望工作氛围温馨，愿意为个性化和情感体验付出时间

### 1.3 核心价值主张
> "一只真正懂你的 AI 猫咪——它观察、它理解、它主动帮忙。"

- **主动感知**: 猫咪持续观察你的工作状态（打字节奏、窗口切换、作息规律），无需你告诉它任何事
- **主动整理**: 自动将打字记录转为可读文本，生成日报、识别待办，信息不再遗漏
- **主动关怀**: 久坐提醒、深夜加班关心、高强度工作后鼓励——在你需要时恰好出现
- **渐进了解**: 猫咪用 3-5 天的自然闲聊逐步了解你，而非一次性问卷轰炸
- **养成乐趣**: 日常使用自然积累好感度，解锁外观/动作/随机惊喜，猫咪和你一起成长
- **社区生态**: 创作分享角色皮肤与装饰，下载他人作品，打造独一无二的猫咪

### 1.4 目标平台
Windows / macOS / Linux（Electron 跨平台）

## 3. 功能规划

### 3.1 宠物养成系统

#### 3.1.1 好感度与等级（增量数值系统）

借鉴 Cookie Clicker（饼干模拟器）的**指数增长 + 转生加成**和 Roblox +1 Speed 的**简单动作 → 数值飙升 → 重生倍率**，以**打字为核心正反馈**的增量数值娱乐系统。

**核心循环：**
```
打字 → 获得亲密度 → 升级(Lv.1→10) → 解锁加成/成就
                                          ↓
                              达到Lv.10 → 转生(可选)
                                          ↓
                              获得❤️羁绊点 → 永久倍率提升
                                          ↓
                              重新从Lv.1开始，但更快 → 再次Lv.10 → 再转生...
```

**设计原则：**
1. **打字 = 主要收入**：高基础、高上限、有心流加成
2. **数值要"滚"**：倍率叠加让同样的行为随时间产出更多
3. **重生不是惩罚**：重置换取永久加成，每轮都更快更爽
4. **生产力不强求**：专注/待办是锦上添花，不是必须

**基础收益（每次行为获得的亲密度）：**

| 行为 | 基础收益 | 每日上限 | 说明 |
|------|---------|---------|------|
| ⌨ 打字 | +1/次按键 | 无上限 | 核心行为，每次键盘按下 +1 |
| ⌨ 心流打字 | +2/次按键 | 无上限 | 连续打字≥5分钟自动触发，收益翻倍 |
| 💬 聊天 | — | — | 更新心情计时（互动），不直接加亲密度 |
| 🐱 摸猫 | — | — | 更新心情计时（互动），不直接加亲密度 |
| 🍅 番茄钟完成 | — | — | 更新心情计时（互动），不直接加亲密度 |
| ✅ 待办完成 | — | — | 更新心情计时（互动），不直接加亲密度 |

> 聊天、摸猫、番茄钟完成、待办完成均调用 `_touch()` 刷新"最后互动时间"，从而保持心情为"开心"（×1.2 倍率），**不直接增加亲密度**。

**倍率系统（最终收益 = 基础 × 总倍率）：**

倍率来源**相乘叠加**（非相加），这是增量游戏数值膨胀的关键：

| 倍率来源 | 数值 | 说明 |
|---------|------|------|
| 转生倍率 | ×(1 + 0.5×转生次数) | 每次转生+50%，核心成长 |
| 心情加成 | 开心×1.2 / 正常×1.0 / 无聊×0.8 | 鼓励保持互动 |
| 连续登录 | ×(1 + 0.02×天数)，上限×2.0 | 最多翻倍 |
| 物品加成 | ×(1 + 道具倍率总和) | 拥有的道具 multiplier additive 叠加 |

**公式：** `实际获得 = 基础收益 × 转生倍率 × 心情倍率 × 登录倍率 × 物品倍率`

**升级曲线：**

| 等级 | 累计亲密度 | 单级所需 | 纯打字天数(估) |
|------|----------|---------|---------------|
| Lv.1 | 0 | — | — |
| Lv.2 | 60 | 60 | ~0.5天 |
| Lv.3 | 180 | 120 | ~1天 |
| Lv.4 | 400 | 220 | ~1.5天 |
| Lv.5 | 700 | 300 | ~2天 |
| Lv.6 | 1100 | 400 | ~2.5天 |
| Lv.7 | 1600 | 500 | ~3天 |
| Lv.8 | 2200 | 600 | ~3天 |
| Lv.9 | 2900 | 700 | ~3.5天 |
| Lv.10 | 3800 | 900 | ~4.5天 |

首次达到Lv.10约需3周正常使用。转生后有倍率加成，第二轮约2周，第三轮约10天...

**转生系统：**
- **条件**：达到 Lv.10
- **重置**：亲密度归零，等级回到 Lv.1，每日统计清零
- **获得**：❤️羁绊点 ×1（永久）+ 额外道具槽位 +1
- **保留**：转生次数、羁绊点、连续登录、心情、已购道具
- **道具槽位**：基础 3 个 + 每次转生 +1（即 `3 + rebirthCount`）

转生倍率：每个羁绊点提供 +50% 总收益
- 0次转生：×1.0（基础）
- 1次转生：×1.5
- 2次转生：×2.0
- 3次转生：×2.5
- ...以此类推，无上限

**心流机制（打字专属加成）：**
- 触发条件：连续打字时长 ≥ 5 分钟（按键间隔 < 2 分钟视为连续）
- 进入心流后，每次按键收益翻倍（+1 → +2 per keystroke）
- UI 表现：打字统计行旁显示 🔥 心流标记
- 猫咪气泡提示："喵~ 你进入心流状态了！亲密度加倍获取中~"

**宠物状态面板 UI：**

转生信息区（在等级行下方）：
```
❤️ 羁绊    第0世 | 当前倍率 ×1.0
[转生] (按钮，Lv.10时亮起)
```

倍率信息区（在日统计下方）：
```
当前倍率明细
  转生加成    ×1.5
  心情加成    ×1.2
  登录加成    ×1.10
  物品加成    ×1.50
  ─────────
  总倍率      ×2.97
```

#### 3.1.2 心情系统 (轻量)
- 三种心情: 开心 / 普通 / 无聊
- 影响因素: 互动频率、连续不操作时间（<5分钟→开心，<20分钟→普通，>20分钟→无聊）
- 表现: 影响猫咪待机动画和随机事件概率
- **倍率影响**: 开心×1.2 / 普通×1.0 / 无聊×0.8（直接影响亲密度获取效率）
- **不设置饥饿/清洁等需要玩家主动维护的负担指标**

#### 3.1.3 随机惊喜事件
- 猫咪随机触发特殊行为（概率受心情和好感度影响）:
  - "猫咪给你叼来了一个小礼物!" → 解锁装饰小物件
  - "猫咪今天特别粘你~" → 特殊对话/表情
  - "猫咪学会了新动作!" → 解锁新 idle 动画
  - "猫咪发现了一个秘密..." → 彩蛋内容
- 每日最多触发 2-3 次，保持稀缺感

#### 3.1.4 外观解锁
- **帽子/饰品**: 等级解锁，叠加在猫咪 sprite 上
- **表情包**: 好感度里程碑解锁新表情动画行
- **特殊皮肤**: 成就/节日限定
- **场景背景**: 猫咪身后可放置小场景(书桌/窗台/花园)

**购买 / 出售规则：**
- 购买消耗亲密度（affinity），亲密度同时作为货币和经验
- 出售回收 40% 原价
- 每件道具占 1 个槽位，槽位上限 = `3 + 转生次数`
- 道具倍率 additive 叠加：`物品倍率 = 1 + Σ(道具.multiplier × 数量)`
- 物品倍率与其他倍率（转生/心情/登录）相乘

#### 3.1.6 联机与排行榜系统

多人联机系统，支持局域网和外部服务器两种模式，可查看其他用户状态和排行榜。

**连接模式：**
- **LAN 内嵌服务器**：Electron 主进程内嵌 WebSocket 服务器（基于 `ws` 库），局域网内其他用户可直接连接
- **外部服务器**：连接远程 WebSocket 服务器（用户自定义 URL），支持跨网络联机

**注册 / 登录：**
- 注册：用户名（2-20 字符）+ 密码
- 密码安全：客户端 SHA-256 预哈希 → 服务端 bcrypt 二次哈希存储
- Token 自动登录：首次登录后保存 token，后续自动登录无需重复输入密码
- 账号数据持久化：服务端 JSON 文件存储（`mp-accounts.json`）

**状态同步协议：**
- 同步字段：`level` / `affinity` / `rebirthCount` / `mood` / `isInFlow` / `skinId` / `totalCPS`
- 速率限制：状态更新最短间隔 5 秒，慢变量（等级/转生）每 30 秒周期同步
- 心跳：30 秒间隔，超时自动断开
- 断线重连：指数退避（1s → 30s 上限），携带 token 自动重新登录

**排行榜：**
- 三种排序维度：按亲密度 / 按等级 / 按转生次数
- 显示所有注册用户（不限于在线），标注在线状态（绿点/灰点）
- 前三名显示奖牌（🥇🥈🥉）

**迷你猫渲染：**
- 桌面上显示其他在线用户的迷你猫咪（emoji sprite：😊/😴/😺）
- 心流状态显示 🔥 徽章
- 最多同时显示 10 只迷你猫

### 3.2 AI 社交系统

#### 3.2.1 AI 猫咪人格
- **性格档案**: 每只猫有基础性格模板(活泼/高冷/软萌/学者)
- **记忆系统**: 记住用户名字、聊天偏好、重要日期
  - 短期记忆: 当前对话上下文
  - 长期记忆: 持久化存储的关键信息摘要(生日/喜好/习惯)
- **情感表达**: 聊天时猫咪表情实时变化（开心/好奇/困惑/撒娇）
- **主动搭话**: 猫咪基于 AI 主动交互系统（详见 3.4）在合适时机发起话题
- **人格影响交互风格**: 同一个主动交互场景，不同性格的猫咪表达方式截然不同：
  - 活泼猫（久坐提醒）: "喂喂喂！你已经坐了 2 小时了！快起来和我一起蹦蹦跳跳！🐾"
  - 高冷猫（久坐提醒）: "……你坐太久了。起来活动一下。别让我担心。"
  - 软萌猫（久坐提醒）: "呜……主人坐了好久了，腰会不会疼呀？陪我去走走好不好嘛~"
  - 学者猫（久坐提醒）: "研究表明，每 90 分钟站立活动 5 分钟可提升 15% 认知效率。建议现在休息。"

#### 3.2.2 社区平台
- **角色/皮肤市场**: 用户上传自制 spritesheet 角色供他人下载
- **装饰物分享**: 分享自制帽子/饰品/场景
- **展示墙**: 截图分享桌面布局
- **评分与热度**: 社区投票排名
- **技术方案**: 独立后端服务(远期)，初期可用 GitHub Release / 静态 CDN

### 3.3 办公生产力工具

#### 3.3.1 番茄钟 / 专注模式
- **标准番茄钟**: 25分钟工作 + 5分钟休息，可自定义时长
- **专注时猫咪行为**: 安静陪伴动画（阅读/打盹），不主动打扰
- **休息提醒**: 猫咪弹出提醒 + 可爱动画
- **统计面板**: 今日/本周番茄数、专注时长趋势图
- **与养成联动**: 完成番茄钟刷新互动计时（保持心情加成）

#### 3.3.2 待办事项 / 提醒
- **快捷添加**: 在聊天中直接说"提醒我下午3点开会" → AI 解析创建待办
- **待办列表**: 轻量 TODO 面板，支持优先级和分类
- **定时提醒**: 到期时猫咪气泡弹出提醒内容
- **每日摘要**: 每天结束猫咪汇报完成情况

#### 3.3.3 打字数据智能 Skill
基于 ChatCat 独有的**打字记录数据源**，通过 AI 自动完成信息整理，无需用户手动操作。

**Skill 1: 打字记录→文本转换**
- **触发频率**: 每 10 分钟自动执行一次
- **功能**: 将原始按键记录还原为可读文本
  - 支持中文拼音输入还原（基于拼音词库 + AI 纠错）
  - 支持英文直接拼接 + 智能断词
  - 自动过滤密码输入框等敏感场景
- **输出**: 存储为 `convertedText_{date}` 供后续 Skill 使用
- **开关**: 用户可在设置中独立启用/禁用

**Skill 2: 信息提取→日报生成**
- **触发频率**: 每天 1 次，可配置触发时间（默认 18:00）
- **功能**: 基于当天的转换文本，AI 自动生成工作日报
  - 提取当日主要工作内容和关键产出
  - 按时间线或主题分类整理
  - 格式化为简洁的 Markdown 日报
- **输出**: 存储为 `dailyReport_{date}`，通过猫咪主动推送通知用户（L3 级别）
- **交互**: 猫咪气泡展示日报摘要，点击可展开信息流面板查看/编辑完整内容

**Skill 3: 信息提取→待办识别**
- **触发频率**: 每 1 小时自动执行一次
- **功能**: 从转换文本中识别潜在的待办事项
  - 识别"TODO"、"待办"、"记得"、"别忘了"等关键词
  - AI 分析上下文判断是否为待办意图
  - 自动去重（与已有待办对比）
- **输出**: 新识别的待办通过猫咪主动推送通知（L2 级别）
- **交互**: 气泡通知中用户可一键确认或忽略

**Skill 调度器 (SkillScheduler)**
- 统一管理所有 Skill 的定时触发
- 支持用户自定义各 Skill 的执行频率
- 支持手动触发任意 Skill
- 执行状态通过猫咪动画反馈（思考中/完成）
- **V1.5 新增**: 从 SkillRegistry 读取 SKILL.md 中的 `schedule` 配置，自动注册定时任务

#### 3.3.4 Skill Agent 系统（V1.5 新增）

基于 SKILL.md 格式的轻量 Agent 系统，复用 CodeBuddy 的 Skill 描述格式。运行时动态读取 `.md` 文件作为 AI prompt 指导任务执行。

**核心原则：**
- 轻量一次性任务，不搞多轮 tool-use 循环
- 省 token：能本地做的不走 AI，关键词匹配优先于 AI 意图识别
- SKILL.md 格式描述 Skill，运行时动态读取 .md 注入 prompt

**SKILL.md 格式（frontmatter 字段）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | Skill 唯一标识 |
| `description` | string | 功能描述 |
| `commands` | string[] | 精确 slash 命令（如 `/report`、`/日报`），0 token 匹配 |
| `keywords` | string[] | 自然语言关键词（如 "生成日报"），regex 匹配，0 token |
| `schedule` | object | 定时触发配置（`cronHour` 或 `interval`） |
| `context` | string[] | 执行时注入的数据源名称（`convertedText`/`todos`/`pomodoroStats`/`quickNotes`） |
| `requiresAI` | boolean | false 的 Skill 走 local-handlers 纯本地逻辑 |
| `localHandler` | string | 当 requiresAI=false 时，指定本地 handler 名 |
| `maxTokens` | number | AI 调用最大 token 数 |
| `temperature` | number | AI 调用温度参数 |

**执行流程：**
```
用户输入 "帮我生成日报" 或 /report
  ↓
SkillRouter (renderer) — 关键词/命令匹配
  ↓ 匹配到 daily-report skill
IPC → main process
  ↓
SkillEngine.execute('daily-report', context)
  ↓
1. 读取 skills/daily-report/SKILL.md body
2. 根据 frontmatter.context 收集数据(打字记录/待办/番茄钟)
3. 拼接 prompt = SKILL.md 指令 + 动态上下文
4. 单次 AI API 调用 (非流式)
  ↓
结果返回 → renderer 渲染到 chat bubble (支持 Markdown)
```

**路由优先级：**
1. **Tier 1**: `/command` 精确匹配（0 token，最高优先级）
2. **Tier 2**: 关键词 regex 匹配（0 token）
3. **Tier 3**: 未匹配 → 正常 AI 聊天

**已内置 Skill：**

| Skill | 类型 | 命令 | 说明 |
|-------|------|------|------|
| daily-report | AI | `/report`、`/日报` | 读取今日打字记录 + 待办 + 番茄钟，AI 生成日报 |
| todo-management | 本地 | `/todo`、`/待办` | 添加/列表/完成/删除待办，0 token |
| note-taking | 本地 | `/note`、`/notes`、`/笔记` | 添加/查看/搜索/删除笔记，0 token |

**架构模块：**

| 模块 | 进程 | 职责 |
|------|------|------|
| `skill-registry.js` | Main | 启动时扫描 `skills/` 目录，解析 SKILL.md frontmatter，缓存元数据 |
| `skill-engine.js` | Main | 执行引擎：读 .md body → 收集上下文 → 调 AI 或本地 handler → 返回结果 |
| `skill-router.js` | Renderer | 拦截 chat 输入，按 /command → keyword 顺序匹配 skill |
| `local-handlers.js` | Main | 笔记/待办等纯本地 store 操作，0 token |

### 3.4 AI 主动交互系统 ⭐ 核心

ChatCat 的核心差异化能力。猫咪不再等待用户指令，而是 **主动观察、主动判断、主动行动**——像一个真正的桌面伙伴。

#### 3.4.1 主动交互引擎 (ProactiveEngine)

**触发信号体系（分版本递进）**

| 版本 | 信号来源 | 具体信号 |
|------|----------|----------|
| V1.2 | 打字数据 + 时间 | 打字速度变化(`typing-speed-change`)、打字停顿(`typing-pause`)、时间点(`time-trigger`)、连续工作时长(`long-work`)、空闲检测(`idle`) |
| V1.5 | 打字节奏 + 剪贴板 + 短空闲 | 打字节奏情感(`typing-rhythm-change`: frustrated/rushing/steady)、工作阶段(`work-phase`: morning-start/afternoon-slump/wrap-up/overtime)、剪贴板内容(`clipboard-content`: url/code/text/repeat)、短空闲(`short-idle`: 5-15min)、转生事件(`prestige`) |
| V2.0 | 窗口识别 + 系统事件 | 前台窗口标题/应用名、窗口切换频率、锁屏/解锁、网络变化、USB 设备插拔、日历事件（需授权） |

**智能时机判断 (TimingJudge)**

- **禁止打扰条件**（命中任一条即抑制推送）:
  - 用户正在连续打字（打字间隔 < 3 秒）
  - 番茄钟专注时段进行中
  - 用户手动开启免打扰模式
  - 距上次推送 < 5 分钟
- **最佳推送时机**（优先在以下时机推送）:
  - 打字停顿 > 15 秒
  - 刚切换窗口（V1.5）
  - 锁屏恢复后（V2.0）
  - 用户刚完成一个番茄钟
- **延迟队列**: 被抑制的推送进入队列，最多缓存 3 条，等到合适时机按优先级依次推送

**分级通知系统 (NotificationMgr)**

| 级别 | 名称 | 表现 | 适用场景 |
|------|------|------|----------|
| L0 | 静默 | 仅记录到信息流，不弹出任何 UI | 低优先级记录（打字速度统计、模式识别结果） |
| L1 | 微标记 | 猫咪身上出现小圆点标记，用户注意到时可点开查看 | 轻量提示（模板建议、小知识分享、高效时段通知） |
| L2 | 气泡 | 猫咪弹出气泡，显示 1-2 行摘要，5 秒后自动缩小为小圆点 | 中等提示（待办识别、休息提醒、进度回顾） |
| L3 | 强调气泡 | 猫咪弹出较大气泡，包含操作按钮，需用户点击响应或手动关闭 | 重要事项（日报生成、待办到期、重要日期祝福） |

#### 3.4.2 渐进式用户画像 (UserProfiler)

**Day 1-5 引导流程**
猫咪用 3-5 天时间，通过自然闲聊逐步了解用户。每天只问 1-2 个问题，伪装为猫咪的好奇心，而非功能性问卷。

## 6. 核心交互流程

### 6.1 首次启动（渐进引导 Day 1）
```
启动 → 猫咪出现在桌面 (idle 呼吸动画)
     → 猫咪气泡: "你好呀！我是你的新猫咪~ 你想叫我什么名字呢？(=^.^=)"
     → 用户输入名字 → 猫咪记住
     → 猫咪: "嘿嘿，谢谢你收养我！对了，你平时是做什么工作的呀？我好奇！"
     → 用户回答 → 记录到 userProfile.occupation
     → 猫咪: "好的好的，记住了！那你先忙吧，我在旁边陪你~有什么都可以点我聊天哦！"
     → 进入日常陪伴状态（不做功能引导，让用户自然探索）
```

### 6.2 日常工作流
```
用户开始打字 → 猫咪打字动画 → 好感度 +1/次按键 (受倍率加成)
连续打字 ≥5分钟 → 进入心流状态 🔥 → 好感度 +2/次按键 (翻倍)
用户点击鼠标 → 猫咪好奇表情（刷新互动计时，不加亲密度）
长时间无操作 → 猫咪打盹/睡觉 → 心情变为无聊(倍率降至×0.8)
用户回来操作 → 猫咪醒来: "你回来啦~" → 心情恢复
达到 Lv.10 → 可选转生 → 获得❤️羁绊点(永久倍率+50%) + 道具槽位+1
```

### 6.3 番茄钟流程
```
点击番茄钟 → 设置时长 → 开始专注
  → 猫咪切换为安静陪伴动画 (阅读/打盹)
  → 计时进行中 (工具栏显示剩余时间)
  → 时间到 → 猫咪弹出: "休息一下吧！你太棒了 ⭐"
  → 刷新互动计时（保持心情为"开心"）
  → 休息计时 → 休息结束 → "准备好继续了吗？加油！"
```

### 6.4 打字 Skill 流程
```
[自动日报]
18:00 到达 → SkillScheduler 触发日报生成
  → 猫咪动画: 思考中...
  → AI 分析当天打字转换文本，生成日报
  → 猫咪 L3 气泡: "今天的工作日报帮你整理好啦！"
  → 气泡展示日报摘要（主要工作 3-5 条）
  → 用户点击 → 展开信息流面板，查看/编辑完整日报

[自动待办识别]
每小时触发 → TodoExtractor 分析最近 1 小时文本
  → 识别到待办: "明天下午 3 点和产品组开会"
  → 猫咪 L2 气泡: "我发现你提到了一个待办哦~"
  → 气泡展示: "明天 15:00 和产品组开会" [✓确认] [✗忽略]
  → 用户确认 → 自动添加到待办列表

[手动触发]
用户在设置面板中点击"立即生成日报"
  → 同自动日报流程
```

### 6.5 主动交互典型流程

**信息整理推送流程**
```
SignalCollector 检测到打字文本中包含待办关键词
  → ProactiveEngine 创建待办识别推送 (场景 2, L2)
  → TimingJudge 检查: 用户是否在连续打字？→ 是 → 进入延迟队列
  → 用户打字停顿 > 15 秒 → TimingJudge 释放队列
  → NotificationMgr 弹出 L2 气泡: "我发现你刚才提到了..."
  → 5 秒后气泡缩小为小圆点
  → 用户稍后点击 → 展开气泡 → 确认待办
  → 记录到 proactiveHistory + feedItems
```

**情绪关怀流程**
```
SignalCollector 检测到连续工作 > 90 分钟（场景 6）
  → ProactiveEngine 创建久坐提醒 (L2)
  → TimingJudge 检查: 番茄钟进行中？→ 否 → 允许推送
  → NotificationMgr 弹出 L2 气泡（根据猫咪性格调整措辞）
  → 活泼猫: "喂喂喂！坐太久啦！起来和我蹦蹦跳跳！"
  → 用户点击"好的" → 刷新互动计时 → 气泡消失
```

**渐进引导 Day 2 流程**
```
用户第二天首次打开 ChatCat
  → 猫咪每日问候 (场景 10): "早上好呀！昨天陪你工作很开心~"
  → 等待用户空闲时机 (停顿 > 15 秒)
  → 猫咪引导式提问: "你一般几点开始工作呀？我好安排我的打盹时间~"
  → 用户回答 → 记录 workSchedule
  → 间隔 30+ 分钟后，另一个空闲时机
  → 猫咪: "你喜欢我经常和你说话，还是安安静静陪着你？"
  → 用户选择 → 记录 interactionPreference → 调整 maxDailyInteractions
```

## 7. UI/UX 设计原则

1. **AI 主动引导优先**: 猫咪是信息和帮助的主动推送者，而非等待用户打开的被动工具箱。所有核心能力通过猫咪主动送达。
2. **气泡为入口，面板为容器**: 气泡通知保持轻量（1-2 行），承载摘要和快捷操作；详细内容展开到信息流面板，避免气泡过载。
3. **尊重用户节奏**: 严格遵守 TimingJudge 规则——用户专注时绝不打扰，空闲时才出现。宁可延迟推送也不打断工作。
4. **渐进式发现**: 新功能通过猫咪主动推荐或等级解锁自然引入，而非一次性暴露所有功能。Day1-5 引导伪装为闲聊而非设置向导。
5. **一致的漫画风格**: 所有面板保持 manga bubble 样式，圆角+尾巴+柔和配色。信息流面板延续同一设计语言。
6. **猫咪人格一致性**: 所有主动推送的文案需匹配猫咪当前性格模板。活泼猫不会说冷淡的话，高冷猫不会用太多语气词。
7. **用户掌控权**: 用户可随时调整主动交互频率、关闭特定场景、设置免打扰时段。所有"主动"都可被用户管控。
8. **跨平台一致性**: Win/Mac/Linux 体验一致，适配各平台系统托盘和快捷键。窗口识别(V1.5)封装平台抽象层。

## 8. 风险与待定事项

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| AI API 成本 | 长期记忆和主动推送增加 token 消耗 | 本地缓存摘要、控制主动推送频率、支持本地模型 |
| spritesheet 美术产能 | 养成解锁需要大量动画素材 | AI 生成 + 社区贡献、模块化 sprite 叠加而非整套重画 |
| 打字隐私 | 打字数据包含用户敏感信息 | 所有数据纯本地存储不上传、密码框自动过滤、用户可随时清除数据、Skill 可独立关闭 |
| 拼音还原准确率 | 拼音→中文还原可能不准确 | 结合词频词库 + AI 上下文纠错、允许用户手动修正、标注置信度 |
| 社区内容审核 | UGC 质量和合规性 | 社区上线前准备审核机制、初期人工审核 |
| Electron 性能 | 多功能导致内存占用升高 | 懒加载模块、idle 时降低渲染帧率、定期优化 |
| 跨平台兼容 | uiohook/系统API平台差异 | 各平台 CI 测试、封装平台抽象层 |
| 主动交互过度打扰 | 用户觉得猫咪太吵而卸载 | 严格的 TimingJudge 规则、分级通知、每日上限(默认8次)、持续监控功能关闭率 |
| 主动交互不够智能 | 推送时机不当或内容无关导致信任下降 | 基于用户响应率动态调整、确认式提问验证 AI 推断、初期保守策略逐步放开 |
| 用户画像数据隐私 | 画像信息涉及用户工作习惯和个人偏好 | 所有画像数据纯本地存储、用户可查看/编辑/删除画像、不上传云端（V2.0 同步需明确授权） |
| 窗口识别隐私(V1.5) | 读取前台窗口标题可能泄露隐私 | 仅读取应用名而非完整标题（可配置）、敏感应用黑名单、用户可关闭窗口识别功能 |
